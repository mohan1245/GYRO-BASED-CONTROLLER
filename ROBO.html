1. Introduction & Motivation

Industrial serial manipulators are traditionally controlled using teach pendants, joysticks, or programmed trajectories. These methods lack intuitive human interaction.

This project proposes a wearable dual-hand gesture-based control system using ESP32 and MPU6050 sensors to enable natural motion-based control of a serial manipulator or robotic system in real-time.

Motivation:

  • Create an intuitive human-robot interface.
  • Reduce dependency on expensive industrial control hardware.
  • Enable immersive robotic manipulation using motion gestures.
  • Bridge embedded systems with robotic kinematics and control.

2. Novelty of the Project

  • Dual independent wearable controllers (one per hand).
  • Real-time UDP-based low-latency wireless communication.
  • Human gesture to manipulator joint-space mapping.
  • Threaded queue-based architecture for high responsiveness.
  • Low-cost alternative to industrial motion control interfaces.

Unlike conventional controllers, this system enables natural Cartesian-space motion mapping to serial manipulator joints.

3. Methodology

3.1 Forward Kinematics

Forward kinematics computes end-effector position from joint variables. We model the serial manipulator using Denavit–Hartenberg (DH) parameters.

T = A1 × A2 × A3 × ... × An

Where Ai represents homogeneous transformation matrices derived from DH parameters. The computed transformation matrix gives:

  • End-effector position (x, y, z)
  • Orientation (Rotation Matrix)

3.2 Inverse Kinematics

Inverse kinematics computes joint angles required to achieve a desired end-effector pose. Gesture inputs are mapped into:

  • Desired Cartesian coordinates
  • Desired orientation adjustments

Geometric and numerical methods are used to solve joint variables:

θ = f⁻¹(x, y, z, orientation)

3.3 Control Strategy

  • MPU6050 detects pitch, roll, yaw.
  • ESP32 processes motion data.
  • UDP protocol transmits encoded motion data.
  • Multi-threaded C program receives data.
  • Queue system processes commands.
  • Joint commands sent to simulated manipulator.

Control type: Real-time position control with gesture-to-joint mapping.

5. Selected Serial Manipulator for Simulation

We simulate a 3-DOF serial manipulator:

  • Revolute Joint 1 – Base rotation
  • Revolute Joint 2 – Shoulder
  • Revolute Joint 3 – Elbow

Simulation Environment:

  • Custom C-based visualization
  • DH parameter modeling
  • Real-time joint updates from gesture inputs

Gesture Mapping Example:

  • Tilt forward → Increase θ2
  • Rotate wrist right → Increase θ1
  • Lift hand → Increase θ3
  • Button press → Gripper close

Result: Smooth manipulator motion corresponding to natural human hand movement.

6. System Architecture

  • MPU6050 (Gyroscope + Accelerometer)
  • ESP32-WROOM Microcontroller
  • UDP Wi-Fi Communication
  • Multi-threaded C Receiver (pthreads)
  • Queue-based Data Processing
  • Manipulator Simulation Module

The architecture ensures:

  • Low latency
  • High responsiveness
  • Separation of data acquisition and execution threads

7. Expected Outcomes

  • Smooth joint-space motion.
  • Low communication delay.
  • Reprogrammable gesture-to-joint mapping.
  • Scalable to 6-DOF industrial arms.
